> **Design System Generator** is a Next.js‐powered toolkit that automates scaffold creation for your custom design system. Generate reusable, themeable components (cards, buttons, forms, etc.) with predefined Tailwind classes, ShadCN UI patterns, and Radix primitives—so you can spin up a consistent UI library in seconds. 🚀

---

## ✨ Features

- **🎨 Component Scaffolding:** Auto-generate React components (Button, Input, Card, etc.) using ShadCN & Radix primitives.
- **🛠️ Theming Support:** Built-in theme generator with Tailwind CSS variables and CSS-vars configuration.
- **🔄 Live Preview:** Preview generated components in a Storybook-like environment (runs on `npm run dev`).
- **📁 File Structure:** Creates a standardized folder structure (`components/`, `styles/`, `hooks/`, `lib/`).
- **🔍 Type Safety:** Preconfigured with Zod validation schemas and TypeScript interfaces.
- **🌐 CLI Utility:** Run `npm run generate` to prompt a wizard for new component templates.

---

## 🛠️ Tech Stack

- **Framework:** Next.js 15 (App Router)  
- **UI Library:** React 19 + ShadCN UI + Radix UI primitives  
- **Styling:** Tailwind CSS 4 + Tailwind Merge + Tailwind CSS Animate  
- **Validation:** Zod & @hookform/resolvers  
- **Language:** TypeScript 5  
- **Bundler/Config:** PostCSS 8, pnpm, Vite (for any standalone CLI modules)  
- **Data Storage (Optional):** Lightweight JSON for storing generated config  

---

## 🚀 Getting Started

### Prerequisites
- Node.js 18+  
- pnpm 7+ (or npm 9+)  

### Quickstart
```bash
git clone https://github.com/your-username/design-system-generator.git
cd design-system-generator
pnpm install
cp .env.example .env.local  # (only if you have environment variables)
pnpm run dev
````

Open [http://localhost:3000](http://localhost:3000) to view the live preview.
To generate a new component scaffold:

```bash
pnpm run generate
```
